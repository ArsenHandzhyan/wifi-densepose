## 📋 Шаг 1: Перезапустите Home Assistant

Поскольку Home Assistant работает не в Docker, перезапустите его одним из способов:

### Вариант A: Через веб-интерфейс
1. Откройте `http://localhost:8123`
2. **Настройки** → **Система** → **Перезагрузка**
3. Нажмите **"Перезагрузить"**

### Вариант B: Через терминал
```bash
# Найдите процесс Home Assistant
ps aux | grep homeassistant

# Перезапустите (команда зависит от способа запуска)
# Например, если через systemd:
sudo systemctl restart home-assistant

# Или если через docker-compose:
cd /Users/arsen/Desktop/wifi-densepose
docker-compose restart homeassistant
```

## 📋 Шаг 2: Добавьте интеграцию

После перезапуска:

1. Откройте Home Assistant: `http://localhost:8123`

2. Перейдите: **Настройки** → **Устройства и службы**

3. Нажмите желтую кнопку **"ДОБАВИТЬ ИНТЕГРАЦИЮ"**

4. В поиске введите: **`Aqara FP2 Cloud`** или **`Aqara FP2`**

5. Выберите интеграцию из списка

## 📋 Шаг 5: Обновите конфигурацию

1. В Home Assistant: **Настройки** → **Устройства и службы**

2. Найдите интеграцию **"Aqara FP2 (europe)"**

3. Нажмите **"Настроить"**

4. В поле **Device ID** вставьте значение из шага 4 (например, `lumi.12345`)

5. Сохраните

## 🔧 Если что-то не работает

### Интеграция не появляется в поиске

1. Проверьте, что файлы на месте:
   ```bash
   ls -la /Users/arsen/Desktop/wifi-densepose/.ha-core/config/custom_components/aqara_fp2/
   ```
   
2. Должны быть файлы:
   - `__init__.py`
   - `manifest.json`
   - `const.py`
   - `config_flow.py`
   - `binary_sensor.py`
   - `sensor.py`

3. Перезапустите Home Assistant еще раз

### Ошибка при настройке

1. Проверьте логи Home Assistant:
   - Настройки → Система → Журналы

2. Проверьте, что токены действительны:
   - Access Token истекает: 2026-03-11
   - Refresh Token истекает: 2026-04-10

### Объекты не обновляются

1. Подождите 30-60 секунд (интервал опроса)
2. Проверьте подключение к интернету
3. Проверьте логи на наличие ошибок API
