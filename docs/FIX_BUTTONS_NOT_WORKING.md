### Причина проблемы:
Браузер кэшировал старую версию `app.js` (v2), в которой **НЕ БЫЛО**:
- Функции `window.t()` для переводов
- Функции `getCurrentLang()` 
- Системы i18n
- Переключателя языка

### Что исправлено:

**Обновлена версия скрипта в index.html:**

```html
<!-- БЫЛО (неправильно): -->
<script src="app.js?v=20260308-v2"></script>

<!-- СТАЛО (правильно): -->
<script src="app.js?v=20260308-v5"></script>
```

## 🔍 ЕСЛИ ВСЁ ЕЩЁ НЕ РАБОТАЕТ

### 1. Очистите кэш браузера

**Mac:** Cmd+Shift+R  
**Windows:** Ctrl+Shift+R

Или:
- DevTools (F12) → Network tab → Disable cache
- Или: Settings → Clear browsing data

### 2. Проверьте версию в браузере

**Откройте исходный код страницы** (Cmd+U или Ctrl+U):

```html
<script type="module" src="app.js?v=20260308-v5"></script>
```

**Должно быть v5!** Если меньше → очистите кэш!

### 3. Проверьте консоль

**DevTools → Console**

Не должно быть ошибок:
```
❌ "window.t is not defined"
❌ "getCurrentLang is not defined"
❌ "applyLanguage is not a function"
```

Если есть → страница загрузила старый кэш!

### 4. Hard Reset

```javascript
// В консоли браузера:
localStorage.clear();
sessionStorage.clear();
location.reload(true);
```

### Почему это важно:

Версия в URL (`?v=20260308-v5`) заставляет браузер:
1. Проверить есть ли новая версия файла
2. Загрузить свежий файл если версия изменилась
3. Не использовать закэшированную старую версию

Без версии или со старой версией:
- Браузер использует кэш (может быть несколько дней/недель)
- Новые функции не загружаются
- Кнопки не работают
