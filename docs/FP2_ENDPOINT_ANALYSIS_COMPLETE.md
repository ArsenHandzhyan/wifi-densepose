# FP2 Endpoint Analysis - Complete Guide

## 🎯 Цель

Провести полную диагностику всех resource endpoint'ов устройства Aqara FP2 для понимания:
- Какие данные реально обновляются при движении
- Какие endpoint'ы существуют но не используются
- Корректно ли система использует все доступные данные

---

## 📋 Доступные скрипты

### 1. **fp2_monitor_all_endpoints.py** (НОВЫЙ, полный мониторинг)

**Что делает:** Мониторит ВСЕ endpoint'ы в реальном времени

**Запуск:**
```bash
cd /Users/arsen/Desktop/wifi-densepose
python3 scripts/fp2_monitor_all_endpoints.py --interval 1.0
```

**Особенности:**
- Показывает ВСЕ ресурсы устройства
- Подсвечивает изменения в реальном времени
- Группирует по важности (🔴🟡⚪)
- Показывает raw координаты
- Определяет новые endpoint'ы

---

### 2. **fp2_resource_diff_scan.py** (Существующий, сравнение)

**Что делает:** Сравнивает два снимка и показывает различия

**Запуск:**
```bash
python3 scripts/fp2_resource_diff_scan.py --output /tmp/fp2_scan.json
```

**Особенности:**
- Делает два снимка с интервалом
- Показывает только изменившиеся
- Сохраняет результат в JSON

---

### 3. **debug_fp2_coordinates.py** (Отладка координат)

**Что делает:** Детальный мониторинг координат и movement events

**Запуск:**
```bash
python3 scripts/debug_fp2_coordinates.py
```

**Особенности:**
- Фокус на координатах (4.22.700)
- Показывает дельту между сэмплами
- Обнаруживает обновления координат

## 🧪 Тестовые сценарии

### Подготовка:

1. **Запустите мониторинг:**
   ```bash

   ```

2. **Подождите базовый снимок** (5-10 секунд)

3. **Начинайте тесты**

---

### Сценарий A: "Полное отсутствие"

**Действия:**
1. Выйдите из комнаты
2. Подождите 30 секунд
3. Наблюдайте за endpoint'ами

**Ожидаемый результат:**
```
⏸️  No changes detected
...
3.51.85         → 0 (Presence (0/1))                    ⚪
13.27.85        → 0 (Movement Event Code)              ⚪
4.22.700        → 0 active targets (Coordinates)       🔴
```

**Вывод:** Базовое состояние без присутствия

---

### Сценарий B: "Вход в зону"

**Действия:**
1. Быстро войдите в detection area
2. Замрите на 5 секунд

**Ожидаемый результат:**
```
⚡ CHANGED RESOURCES (3):
  🔴 3.51.85: 1 (Presence (0/1))                      ← Изменился!
  🔴 13.27.85: 1 (Static presence)                    ← Изменился!
  🔴 4.22.700: 1 targets: [(49, 330)]                 ← Появилась цель!
```

**Вывод:** FP2 обнаружил присутствие

---

### Сценарий C: "Микро-движения" (Дыхание)

**Действия:**
1. Стойте неподвижно
2. Глубоко дышите
3. слегка двигайте руками

**Ожидаемый результат:**
```
⚡ CHANGED RESOURCES (1):
  🔴 13.27.85: 2 (Micro-movement)                     ← Дыхание!
  ⚪ 4.22.700: 1 targets: [(49, 330)] (no change)     ← Координаты стоят

**Вывод:** FP2 видит микро-движения, но координаты не меняются

### Сценарий D: "Активное движение" (Хождение)

**Действия:**
1. Ходите перед сенсорром слева направо

3. Машите руками

**Ожидаемый результат:**
```

  🔴 13.27.85: 7 (Moving)                             ← Движение!

**Вывод:** FP2 обновляет координаты при макро-движении

## 📊 Анализ результатов

### Известные endpoint'ы FP2:

#### Обязательные (обновляются всегда):

| Resource ID | Название | Тип | Описание |
|-------------|----------|-----|----------|
| `3.51.85` | Presence | Boolean | 0/1 - присутствие обнаружен |
| `0.4.85` | Light Level | Integer | Люксы (0-1000+) |
| `8.0.2026` | RSSI | Integer | Уровень сигнала dBm (-30 to -90) |
| `8.0.2045` | Online State | Boolean | 0/1 - онлайн ли устройство |
| `13.27.85` | Movement Event | Integer | Код события движения (0-10) |
| `4.31.85` | Fall State | Integer | Код падения (0-2) |
| `8.0.2116` | Sensor Angle | Float | Угол сенсора (0-360°) |
| `13.120.85` | Total Count | Integer | Общее количество целей |
| `4.22.700` | Coordinates | JSON | Массив координат целей |

#### Зонные (обновляются при occupancy):

| Шаблон | Название | Пример | Описание |
|--------|----------|--------|----------|
| `3.{N}.85` | Zone Occupancy | `3.1.85` | Занята ли зона N (0/1) |
| `13.12{N}.85` | Zone Target Count | `13.121.85` | Сколько целей в зоне N |

Где N = номер зоны от 1 до 30

#### Специальные (могут существовать):

- `8.0.2008` - Battery level (если беспроводное)
- `3.15.85` - Unknown motion sensitivity
- `13.58.85` - Unknown diagnostic

## 🔍 Диагностика проблем

### Проблема 1: "Координаты не обновляются"

**Симптомы:**
```
Movement Event: 7 (Moving)
Coordinates: [(49, 330)] - не меняется 30s+
Delta: +0, +0
```

**Проверка:**
```bash
python3 scripts/debug_fp2_coordinates.py
```

**Возможные причины:**
1. Aqara Cloud EU ограничивает частоту
2. Микро-движения (недостаточно для обновления)
3. "Призрачные" цели (FP2 запомнил старые)

**Решение:**
- Использовать Movement Event как индикатор
- Перезагрузить FP2 (выключить из розетки на 10с)
- Проверить чувствительность в Aqara Home app

---

### Проблема 2: "Зоны не работают"

**Симптомы:**
```
3.1.85 → нет изменений
13.121.85 → нет изменений
```

**Причины:**
1. Зоны не настроены в Aqara Home
2. Устройство не поддерживает зонирование
3. Неправильный номер зоны

**Проверка:**
```bash
python3 scripts/fp2_monitor_all_endpoints.py | grep "3\..*\.85"
```

**Решение:**
- Настроить зоны в Aqara Home app
- Проверить модель устройства (PS-SO2RU поддерживает зоны)

---

### Проблема 3: "RSSI не обновляется"

**Симптомы:**
```
8.0.2026 → -73 dBm (не меняется минуты)
```

**Это нормально!**
- RSSI обновляется редко (при изменении качества связи)
- Не является индикатором движения

## 🛠️ Дополнительные команды

### Получить все endpoint'ы одним запросом:

```bash
curl -s http://127.0.0.1:8000/api/v1/fp2/current | \
python3 -c "import sys,json; d=json.load(sys.stdin); \
rv=d['metadata']['raw_attributes'].get('resource_values',{}); \
[print(f'{k}: {v}') for k,v in sorted(rv.items())]"
```

### Сравнить два снимка:

```bash
# Snapshot 1
curl -s http://127.0.0.1:8000/api/v1/fp2/current > /tmp/snap1.json

# Подождать 5 секунд

# Snapshot 2
curl -s http://127.0.0.1:8000/api/v1/fp2/current > /tmp/snap2.json

# Сравнить
diff <(cat /tmp/snap1.json | python3 -m json.tool) \
     <(cat /tmp/snap2.json | python3 -m json.tool)
```

### Логировать изменения:

```bash
python3 scripts/fp2_monitor_all_endpoints.py 2>&1 | \
tee >(grep "CHANGED" >> /tmp/fp2_changes.log)
```
