## Что ограничено Open API / текущим транспортом

Это важно не скрывать, а явно отмечать:

### Недоступно или не подтверждено публичным Aqara Open API
- полноценный редактор карты с мебелью и готовыми шаблонами как в приложении;
- запуск AI Learning;
- управление AI body recognition как гарантированно подтвержденный отдельный облачный ресурс;
- экран `Найти устройство`;
- экран `Настройки главной страницы`;
- сетевые/плагин-метаданные уровня приложения Aqara;
- полная историческая аналитика Aqara за прошлые дни, если она не была накоплена локально у нас заранее.

### Причина
- Aqara Home использует более богатый внутренний/приватный набор данных и экранов, чем публичный Open API;
- наш UI строится на том, что реально приходит от FP2 через `Aqara Cloud` и локальный backend;
- часть функций Aqara Home — это UI/управление приложения, а не телеметрия устройства как таковая.

## Практический план parity

### Уровень 1. Полный parity по данным
- обработать все `resource_values`, которые приходят в live payload;
- вынести значимые каналы из raw grid в отдельные UI-блоки;
- улучшить журнал событий до формулировок Aqara Home.

### Уровень 2. Полный parity по обзору
- доделать обзорные карточки и day/week графики;
- связать их с локально накапливаемой историей;
- добавить компактный домашний summary-блок в Dashboard.

### Уровень 3. Partial parity по настройкам
- показать все наблюдаемые настройки FP2 в структурированном виде;
- явно разделить:
  - доступно через API
  - только чтение
  - только в приложении Aqara

### Уровень 4. Честный parity-gap
- для недоступных через API функций оставлять информативные read-only секции, а не молчание;
- если позже появится локальный/private transport, подключать их уже поверх этого слоя.
