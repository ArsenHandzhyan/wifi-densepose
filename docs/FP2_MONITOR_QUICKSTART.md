## 🎯 ЧТО ВЫ УВИДИТЕ

### Когда начнёте двигаться:

```
================================================================================
📊 Замер #5 @ 03:15:30.456
================================================================================

⚡ ИЗМЕНИЛИСЬ РЕСУРСЫ (3):
  🔴 ВАЖНО 13.27.85: 7 (Код события движения)
  🔴 ВАЖНО 4.22.700: 2 цель(ей): [(52, 335), (23, 315)]
      Сырые: [{"rangeId":0,"x":52,"y":335,"state":"1"}, ...]
  🟡 ЗОНА 3.1.85: Зона 1: ЗАНЯТА
```

### Расшифровка:

- **🔴 ВАЖНО** - Критические изменения
  - `3.51.85` - Присутствие (0/1)
  - `13.27.85` - Код движения (0-10)
  - `4.22.700` - Координаты целей

- **🟡 ЗОНА** - Изменения зон
  - `3.X.85` - Занятость зоны X
  - `13.12X.85` - Количество целей в зоне X

- **⚪** - Остальные данные
  - RSSI, свет, угол сенсора

## 🧪 ТЕСТОВЫЕ СЦЕНАРИИ

### Тест 1: Вход в комнату
1. Выйдите из комнаты (30 секунд)
2. Быстро войдите обратно
3. **Ожидайте:** `3.51.85: 0 → 1`, `13.27.85: 0 → 1`

### Тест 2: Микро-движения
1. Замрите
2. Глубоко дышите
3. Двигайте пальцами
4. **Ожидайте:** `13.27.85: 1 → 2` (Micro-movement)

### Тест 3: Активное движение
1. Ходите из стороны в сторону
2. Машите руками
3. **Ожидайте:** `13.27.85: 7` (Moving), `4.22.700` обновляется

### Тест 4: Уход
1. Быстро выйдите из комнаты
2. **Ожидайте:** `3.51.85: 1 → 0`, `13.27.85: 6 → 0`

## 💡 СОВЕТЫ

1. **Записывайте результаты**
   - Сохраните вывод: `python3 scripts/fp2_monitor_ru.py | tee /tmp/fp2_test.log`

2. **Проверьте все зоны**
   - Если настроены зоны, смотрите `3.*.85` и `13.12*.85`

3. **Следите за координатами**
   - Endpoint `4.22.700` показывает как часто обновляются координаты
   - Сравните с Movement Event `13.27.85`

4. **Используйте разный интервал**
   - Быстрее: `--interval 0.5`
   - Медленнее: `--interval 3.0`
